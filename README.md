# Accounts_list
У учетной записи (класс Account) есть имя (поле name) и электронный ящик (поле email). Нужно, чтобы метод getEmails(ArrayList<Account>) в классе Solution возвращал поток электронных ящиков всех учетных записей из списка, полученного в качестве входящего аргумента.

Подсказка:
Для преобразования потока данных одного типа в другой используй метод map() объекта типа Stream<Account>.

Метод main() не принимает участие в тестировании.


Псевдокод:

```
ПРОГРАММА Преобразование данных

КЛАСС Solution:
    МЕТОД main(args):
        СОЗДАТЬ accounts как пустой список Account
        
        ДОБАВИТЬ в accounts:
            новый Account("Elly", "elly@yandex.ru")
            новый Account("Joker", "jocker@gmail.com")
            новый Account("Neo", "neo@yahoo.com")
        
        ПРИСВОИТЬ emails = getEmails(accounts)
        ВЫЗВАТЬ для каждого элемента в emails: печать элемента
    
    МЕТОД getEmails(accounts - список Account) ВОЗВРАЩАЕТ поток строк:
        ВОЗВРАТИТЬ accounts.получитьПоток()
                   .преобразовать(каждый account -> account.получитьEmail())

КЛАСС Account:
    ПОЛЯ:
        name - строка
        email - строка
    
    КОНСТРУКТОР Account(имя, email):
        this.name = имя
        this.email = email
    
    МЕТОД getName() ВОЗВРАЩАЕТ строку:
        ВОЗВРАТИТЬ name
    
    МЕТОД getEmail() ВОЗВРАЩАЕТ строку:
        ВОЗВРАТИТЬ email
```

Описание логики:

1. main метод:
   · Создает список учетных записей (Account)
   · Добавляет три тестовые учетные записи
   · Получает поток email-адресов через метод getEmails
   · Выводит каждый email на экран
2. getEmails метод:
   · Преобразует список Account в поток (Stream)
   · С помощью операции map преобразует каждый объект Account в его email (строку)
   · Возвращает поток строк (email-адресов)
3. Класс Account:
   · Простой класс-модель с двумя полями: имя и email
   · Имеет геттеры для доступа к полям

Ключевые моменты:

· Используется Stream API для функциональной обработки коллекций
· map операция преобразует один тип данных в другой
· Метод getEmails не обрабатывает null-значения (в данном примере их нет)
· Все email выводятся на консоль построчно
